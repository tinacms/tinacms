# Hugo Kitchen-Sink

The Hugo version of the TinaCMS kitchen-sink example app. Built with Hugo Extended and Tailwind CSS 4, demonstrating all core TinaCMS features ‚Äî admin panel editing, block-based pages, rich-text content, and multi-collection schemas ‚Äî as a fully static site with zero client-side JS in production.

## Architecture

Two-layer model:
1. **Hugo templates** (`layouts/`) ‚Äî file-based routing, data fetching via Hugo's `site.GetPage`, static generation
2. **TinaCMS admin panel** (`/admin/`) ‚Äî React-based editing UI, auto-generated by TinaCMS at build/dev time

Unlike the Next.js and Astro versions, Hugo cannot use `useTina()` for visual/inline editing. All content editing happens via the admin panel.

## Collections

6 TinaCMS collections:
- **Tag** (`tina/collections/tag.ts`) ‚Äî simple JSON tags (css, html, react)
- **Author** (`tina/collections/author.tsx`) ‚Äî markdown authors with avatar, description, hobbies
- **Global** (`tina/collections/global.ts`) ‚Äî site-wide config (header nav, footer social links, theme settings)
- **Post** (`tina/collections/post.tsx`) ‚Äî markdown posts with excerpt, author reference, tags, nested folder support
- **Blog** (`tina/collections/blog.tsx`) ‚Äî markdown blog entries with hero image, pub/updated dates, author reference
- **Page** (`tina/collections/page.tsx`) ‚Äî markdown pages with block-based content (hero, features, CTA, testimonial, content blocks)

## Key Patterns

- **Admin-only editing:** Content is edited via TinaCMS admin UI at `/admin/`. No inline/visual editing ‚Äî that requires React `useTina()` which Hugo cannot use.
- **`localContentPath`:** TinaCMS reads content from `examples/shared/` via `localContentPath: '../../../shared'` in tina config. No content symlinks needed.
- **Hugo static folder:** Hugo uses `static/` (not `public/`). TinaCMS config sets `build.publicFolder: 'static'` and `media.tina.publicFolder: 'static'`.
- **Image symlinks:** `static/uploads` and `static/blocks` are symlinked to `examples/shared/public/` for shared assets.
- **Data-only collections:** Tag and Global collections use `build: {render: never, list: never}` in Hugo `_index.md` files to prevent page generation. Data is accessed via `site.GetPage`. Note: use `build` not `_build` ‚Äî the `_build` key was deprecated in Hugo 0.145.0.
- **Content format:** Hugo uses `.md` (not `.mdx`). TinaCMS collection configs set `format: 'md'`. Rich-text body is rendered via Hugo's Goldmark markdown renderer, not TinaMarkdown.
- **Non-body rich-text fields:** Fields like `excerpt` and hero `text` use `string` type with `textarea` UI instead of `rich-text`, since Hugo cannot render rich-text AST stored in frontmatter.
- **Reference resolution:** TinaCMS stores references as path strings (e.g., `content/authors/napoleon.md`). Hugo templates resolve these using `site.GetPage`.
- **Block rendering:** Page collection uses block-based content. Hugo templates dispatch to block partials via `{{ partial (printf "blocks/%s.html" ._template) . }}`.

## Build & Dev

- **Dev:** `pnpm dev` ‚Üí `tinacms dev -c "hugo-extended server"` (TinaCMS wraps Hugo)
- **Build:** `pnpm build` ‚Üí `tinacms build && hugo-extended`
- **Build local:** `pnpm build:local` ‚Üí `tinacms build --local --skip-cloud-checks -c "hugo-extended"`
- **Hugo-only:** `pnpm serve` / `pnpm build:hugo`

## Routes

| Route | Template | Description |
|-------|----------|-------------|
| `/` | `index.html` | Home page with block-based content |
| `/posts/` | `posts/list.html` | Post listing |
| `/posts/:slug` | `posts/single.html` | Post detail (supports nested paths) |
| `/blog/` | `blogs/list.html` | Blog listing (URL overridden to singular `/blog/`) |
| `/blog/:filename` | `blogs/single.html` | Blog detail |
| `/authors/` | `authors/list.html` | Author listing |
| `/authors/:filename` | `authors/single.html` | Author detail |
| `/admin/` | `static/admin/index.html` | TinaCMS admin panel |

## Formatting

This project uses **Biome** for code formatting. Run `npx @biomejs/biome format --write <file>` after creating or modifying files.

## Tailwind Theme

Theme colors use CSS custom properties driven by a `data-theme` attribute on the root div. The `tailwind.config.js` maps theme color utilities to CSS variables. The `data-theme` value is set from the Global collection's theme settings. Dark mode uses Tailwind's `dark:` variant with class-based toggling.

## Shared Content

Content files live in `examples/shared/content/` (single source of truth for all kitchen-sink projects). This project uses `localContentPath: '../../../shared'` in `tina/config.tsx` so TinaCMS reads content directly from the shared directory.

Static assets (`static/uploads`, `static/blocks`) are symlinked to `examples/shared/public/`.

## Known Limitations vs Next.js/Astro Versions

| Feature | Next.js/Astro | Hugo |
|---------|---------------|------|
| Visual/inline editing | Click-to-edit via `useTina()` | Admin panel only |
| Image optimization | Framework `<Image>` component | Standard `<img>` tags |
| Client-side navigation | SPA-style | Full page reloads |
| Custom rich-text templates | BlockQuote, DateTime, NewsletterSignup | Not supported (Goldmark) |
